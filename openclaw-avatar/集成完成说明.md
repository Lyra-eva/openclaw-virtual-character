## 🚀 使用流程

### 第一次使用

```bash
# 1. 安装依赖
npm install --legacy-peer-deps

# 2. 下载模型
npm run download-models

# 3. 启动应用
npm run tauri:dev
```

1. **打开配置面板** - 点击底部 ⚙️ 按钮

3. **下载模型** - 点击"📥 下载模型"
4. **初始化** - 点击"⚡ 初始化"
5. **测试** - 点击"🔊 测试 TTS"
6. **开始使用** - 关闭配置面板，点击 🎙️ 按钮说话

## 🔧 配置说明

### 模型选择

| 模型 | 大小 | 适用场景 |
|------|------|----------|
| Tiny | 39MB | 快速响应，精度要求不高 |
| Base | 74MB | ⭐ 平衡，推荐使用 |
| Small | 244MB | 高精度，嘈杂环境 |

### TTS 语速

- **0.5x - 0.8x**: 慢速，适合学习/演示
- **0.9x - 1.1x**: 正常，日常对话
- **1.2x - 1.5x**: 快速，提高效率
- **1.6x - 2.0x**: 极快，特殊需求

### VAD 灵敏度

- **10-30%**: 极灵敏，安静环境
- **31-50%**: 正常，一般环境 ⭐
- **51-70%**: 较低，嘈杂环境
- **71-90%**: 低，非常嘈杂

## 🐛 已知问题

### 轻微问题

1. **首次加载慢**
   - 原因：模型文件较大 (200MB+)
   - 解决：一次性下载，后续缓存

2. **CPU 占用波动**
   - 原因：Whisper 推理时占用较高
   - 解决：使用 Tiny 模型或降级到 Web Speech

3. **唇形同步延迟**
   - 原因：音频分析需要时间
   - 解决：调整 `smoothingFactor` 参数
